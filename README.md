# Tools

## copy_backup.sh 使用指南

`copy_backup.sh` 是一支用來批次更新目標資料夾內專案壓縮檔的工具。在進行更新之前，它會先為目標資料夾中與壓縮檔同名的資料夾建立時間戳記備份，並於備份完成後詢問使用者是否要以壓縮檔內容覆蓋原資料。

### 功能特色
- **自動備份**：針對每個同名資料夾產生 `ZIP` 壓縮備份，備份檔案名稱包含時間戳記，儲存在指定的備份路徑中。
- **互動式確認**：備份完成後會提示使用者是否要進行取代，只有在輸入 `Y`/`y` 時才會複製並解壓縮新的壓縮檔。
- **dry-run 模式**：透過 `--dry-run` 參數可以預覽整個流程，不會對檔案系統進行任何修改。
- **路徑預設/自訂**：可直接指定目標與備份路徑；若未提供，腳本會詢問是否使用預設路徑，或由使用者輸入。

### 依賴套件
請先確認系統已安裝下列指令：

- `zip`
- `unzip`

若缺少可透過系統套件管理工具安裝，例如：

```bash
sudo apt install zip unzip
```

### 準備工作
1. 將欲更新的壓縮檔（副檔名 `.zip`）放在 `copy_backup.sh` 同一個資料夾內，例如 `Tools/script/`。
2. 確認目標路徑與備份路徑存在，或讓腳本自動建立。

### 指令語法
```bash
./copy_backup.sh [--dry-run] [目標路徑] [備份路徑]
```
- `--dry-run`：選擇性參數，僅顯示會執行的操作，不實際執行。
- `目標路徑`：選擇性參數，要更新的資料夾所在位置。預設為 `/home/threedlidar/IUMOBO_ws/src`。
- `備份路徑`：選擇性參數，備份壓縮檔要儲存的位置。預設為 `/home/threedlidar/Backup`。

若未同時提供 `目標路徑` 與 `備份路徑`，腳本會詢問是否使用預設路徑；若回答 `N`，則會互動式要求輸入兩個路徑。

### 執行流程說明
1. **啟動與檢查**：腳本會檢查必要指令，並建立（如需）目標與備份目錄。
2. **搜集待處理檔案**：讀取腳本所在資料夾內所有 `.zip` 檔案；若沒有檔案會顯示警告並結束。
3. **備份既有資料夾**：對於每個壓縮檔 `xxx.zip`，若目標路徑存在 `xxx` 目錄，會先將其壓縮備份為 `備份路徑/xxx_YYYYMMDD-HHMM.zip`，然後刪除原資料夾。
4. **確認是否取代**：備份完成後會詢問是否要進行取代。
   - 輸入 `Y`/`y`：複製每個 `.zip` 到目標路徑、解壓縮並移除壓縮檔，最後顯示完成訊息。
   - 其他輸入：不進行取代，流程即刻結束。

### 使用範例
```bash
cd Tools/script
# 實際執行，使用預設路徑
./copy_backup.sh

# 模擬執行，指定自訂路徑
./copy_backup.sh --dry-run /path/to/target /path/to/backup
```

### 注意事項
- 建議在執行前確認目標路徑中的資料已停止使用，以避免競爭狀況。
- 備份檔案會不斷累積，請定期清理備份資料夾。
- 若腳本所在資料夾同時擁有多個 `.zip`，會依檔名逐一執行上述流程。
